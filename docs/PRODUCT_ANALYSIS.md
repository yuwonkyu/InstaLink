## 1. 서비스 현황 요약

**구현 완료 (릴리스 가능 수준)**
- 다중 테마 공개 프로필 페이지 (`/[slug]`)
- Supabase Auth 기반 회원가입/로그인
- 대시보드 편집 (탭 5개: 기본·디자인·서비스·콘텐츠·고급)
- 토스페이먼츠 구독 결제 (Free / Basic / Pro)
- 링크 클릭 추적 + 방문자 통계 기본 구조
- 레퍼럴 코드, QR 코드, 후기 수집 링크
- 관리자 대시보드 (`/admin`)
- SEO — 업종별 카테고리 페이지 7개, JSON-LD, sitemap

**미완성 / 플레이스홀더**
- 일별 방문자 차트: `app/dashboard/page.tsx` 269번 줄 → "곧 제공 예정" 텍스트만 표시
- `error.tsx` 에러 바운더리: 대부분 라우트에 없음 (DB 오류 시 흰 화면)
- 서버사이드 입력 검증: 카카오 URL·인스타 ID 형식 검사 없음
- 결제 실패 자동 재시도: 수동 재시도 API만 있고 스케줄 없음

## 2. UX/UI 문제점 (심각도 순)

### 🔴 Critical — 즉시 수정 필요

#### C-1. 무료 플랜 갤러리 0장
- 위치: `lib/plan-limits.ts`의 Free 플랜 갤러리 제한
- **문제**: 인스타그램 마케팅하는 소상공인이 포트폴리오 사진 없는 페이지를 쓸 이유가 없다.
- **해결**: Free → 갤러리 3장으로 상향. 첫 가치 경험 없이는 전환 불가.

#### C-2. 공개 페이지 CTA 시각적 위계 없음
- 위치: `components/ProfilePage.tsx` CTA 버튼 섹션
- **문제**: 카카오·인스타·전화가 동일 크기로 나열 → 사용자가 어디를 눌러야 할지 모름.
- **해결**: 메인 CTA(카카오 문의) 1개 full-width + 보조 CTA 소형 아이콘 버튼으로 분리.

#### C-3. 바이럴 배지 없음
- **문제**: 공개 페이지 하단에 "InstaLink로 만들기" 배지가 없어 유기적 성장 루프 없음.
- **해결**: 공개 페이지 푸터에 "인스타링크로 무료 만들기" 배지 추가. Pro 플랜에서만 숨기기 가능.

### 🟡 Major — 1~2주 내 처리

#### M-1. 온보딩 마찰
- 위치: `app/dashboard/onboarding/page.tsx`
- **문제**: 이메일 인증 대기 → 온보딩 입력 → 대시보드 순서로 가입 후 첫 가치 도달이 느림.
- **해결**: 슬러그 + 이름 + 카카오 URL 3가지만 받고 즉시 페이지 생성. 나머지는 대시보드에서 편집.
- **목표**: 가입 후 3분 안에 공유 가능한 링크 생성 완료.

#### M-2. 대시보드 탭 구조 과복잡
- 위치: `app/dashboard/edit/page.tsx`, `components/EditForm.tsx`
- **문제**: 5개 탭(기본·디자인·서비스·콘텐츠·고급)은 소상공인에게 압도적.
- **해결**: 탭 4개로 병합.
  ```
  현재: 기본 | 디자인 | 서비스 | 콘텐츠 | 고급
  목표: 내 페이지 | 서비스·메뉴 | 사진·후기 | 설정
  ```

#### M-3. 실시간 편집 미리보기 없음
- **문제**: 편집 저장 후 새 탭에서 확인해야 함 → 편집 경험 나쁨.
- **해결**: 대시보드 오른쪽에 iframe 미리보기 패널 추가 (데스크탑). 모바일은 "미리보기" 버튼으로 임시 팝업.

#### M-4. 일별 방문자 차트 미구현
- 위치: `app/dashboard/page.tsx:269`
- **문제**: Pro 유저가 결제를 유지할 핵심 이유인데 플레이스홀더만 있음.
- **해결**: `page_views` 테이블 날짜별 집계 → 14일 라인 차트 구현.

### 🟢 Minor — 이후 처리

- 에러 바운더리 추가 (`error.tsx`) — DB 오류 시 흰 화면 방지
- 입력 검증 강화 — 카카오 URL, 인스타 ID 형식 서버사이드 검사
- 결제 자동 재시도 스케줄 — 실패 후 3일, 7일 재시도 Vercel Cron

## 3. 마케팅 분석

### 경쟁 포지셔닝

| 서비스 | 가격 | 특징 | InstaLink 차별점 |
|--------|------|------|-----------------|
| Linktree | $0~$24/월 | 글로벌·단순 링크 | 한국 소상공인 특화, 카카오·네이버 연동 |
| 링크인바이오 | 무료 | 한국어 | InstaLink가 기능 훨씬 많음 |
| 아임웹 | 39,900원~/월 | 풀 페이지 빌더 | InstaLink가 훨씬 간단하고 저렴 |
| 카카오채널 | 무료 | 메시지 중심 | 보완 관계 (카카오채널 연동) |

**결론: 경쟁이 없는 틈새 — 단 아무도 모름.**

### 핵심 마케팅 문제

#### 1. 타겟이 너무 분산
- 현재 SEO 카테고리 7개 동시 공략 = 어느 쪽도 강력하지 않음.
- **1차 집중 타겟: PT 트레이너**
  - 인스타 마케팅 의존도 최고
  - 개인 브랜딩 욕구 강함
  - 가격 민감도 낮음 (월 29,000원은 PT 1회 금액)
  - 트레이너 오픈채팅·카페 커뮤니티 존재

#### 2. 전환 퍼널 병목
현재 퍼널: `랜딩 → 회원가입` (중간 단계 없음)

권장 퍼널:
```
인스타 광고/바이럴
  → 랜딩페이지
  → 로그인 없이 데모 편집 체험 (인터랙티브 샘플)
  → "내 버전 만들기" 클릭
  → 간단 가입 (3필드)
  → 즉시 완성 페이지
  → 공유 → 고객 유입 확인
  → 14일 후 업그레이드 유도
```

#### 3. 바이럴 루프 미작동
- 공개 페이지에 배지가 없으면 Linktree식 바이럴 성장 불가.
- 소상공인 A 페이지를 본 고객 B(다른 사장)가 "나도 이런 거" → 배지 클릭 → 가입.

#### 4. B2B 채널 미개발 (가장 빠른 CAC 감소 수단)
- 인스타 마케팅 강사 (유튜브, 클래스101) → 제휴 수수료 + 할인 코드
- 소상공인 컨설턴트 → 번들 추천
- 소상공인진흥공단 파트너 등록

### 가격 전략 권장

| 현재 | 권장 | 이유 |
|------|------|------|
| Free: 갤러리 0장 | Free: 갤러리 3장 | 첫 가치 경험 필수 |
| 14일 통계 무료 | 14일 Pro 무료 체험 | 끝나도 데이터 유지, 결제 유도 |
| - | 연간 할인 배너 | 월간 결제자 전환 유도 |

## 4. PMF 가능성 판단

**긍정 요소**
- 명확한 타겟 페르소나 존재
- 실제 Pain Point (링크트리는 한국 소상공인에 너무 범용)
- 결제·인증·편집 기술 스택 완비
- SEO 기반 구축

**위험 요소**
- 레퍼런스 고객 없음 (실제 매출 발생 사례 0)
- 무료 플랜 가치 부족 (갤러리 0장)
- 온보딩 마찰 높음
- 바이럴 루프 없음

**판단**: 기술 준비도 70/100 · 시장 준비도 45/100 · 마케팅 준비도 30/100

> 지금 해야 할 가장 중요한 것:  
> 코드보다 **PT 트레이너 5명에게 직접 써달라고 부탁하고 어디서 막히는지 관찰**.  
> 첫 고객을 무료로 세팅해주고 피드백 받는 것이 가장 빠른 PMF 검증.

## 5. Claude 작업 원칙 (이 분석에서 도출)

작업 우선순위를 결정할 때 아래 기준을 따른다:

1. **가입→활성 전환에 영향 주는 것** 최우선 (C-1, C-2, C-3, M-1)
2. **Pro 결제 유지에 영향 주는 것** 두 번째 (M-4 통계 차트)
3. **편집 경험 개선** 세 번째 (M-2, M-3)
4. **안정성·검증** 마지막 (에러 바운더리, 입력 검증)

기능 추가보다 **기존 기능의 전환율 개선**이 항상 우선이다.
